# Orchestration Contract

> Authoritative default generated by ADOm8 onboarding. Repository owner may supersede this file intentionally.

## Stage Order

`Planning → Coding → Testing → Review → Documentation → Deployment`

## InitializeCodebase Exception

InitializeCodebase stories are a special no-clone path:
- Routed directly to Coding
- Review/Testing/Documentation/Deployment queue stages may be skipped
- Story is moved to Code Review after Copilot reconciliation

## Required ADO Update Protocol

At each completed stage, update Azure DevOps with all of:
1. Work item state update when stage boundary requires it
2. Work item comment with concrete evidence (plan, PR, tests, review outcome, deployment)
3. Stage event record for traceability

When MCP bridge endpoints are configured, use:
- `set-stage`
- `add-comment`
- `stage-event`

## Idempotency Rules

- Stage event submissions must be replay-safe.
- Re-running the same stage must not duplicate destructive actions.
- PR linking and progress reporting calls should be safe to repeat.

## PR and Branch Expectations

- Start from the story base branch provided in the issue body.
- Open one PR per story task.
- Include work item ID (`US-{id}`) in PR metadata for reconciliation.

## Security Guardrails

- Never commit secrets or credentials.
- Follow repository-specific security rules in `AGENTS.md` when present.
- Escalate if a requested change conflicts with compliance or policy constraints.
